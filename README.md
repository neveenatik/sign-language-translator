# 🤟 Real-time ASL to Text Translator

A Python application that translates American Sign Language (ASL) to text in real-time using your webcam. The system uses MediaPipe for hand pose detection and TensorFlow for gesture recognition.

## Features

- ✅ **Real-time Translation**: Live webcam feed with instant ASL recognition
- ✅ **Hand Landmark Detection**: Accurate hand pose tracking using MediaPipe
- ✅ **Gesture Recognition**: Deep learning model for ASL letter classification
- ✅ **Temporal Smoothing**: Reduced noise through prediction smoothing
- ✅ **Easy Training**: Scripts to train custom models on your own ASL data
- ✅ **Web Interface**: Streamlit-based UI for easy interaction

## Quick Start

### Prerequisites

- Python 3.8+
- Webcam (for live translation)
- 4GB+ RAM recommended

### Installation

1. Clone the repository:
```bash
git clone https://github.com/neveenatik/sign-language-translator.git
cd sign-language-translator
```

2. Create a virtual environment:
```bash
python3 -m venv venv
source venv/bin/activate
```

3. Install dependencies:
```bash
pip install -r requirements.txt
```

### Running the Application

Start the Streamlit app:
```bash
streamlit run app.py
```

The app will open in your browser at `http://localhost:8501`

## How It Works

### 1. Hand Pose Detection

MediaPipe Hands detects 21 landmarks per hand:
- Wrist, palm, and finger joints
- 3D coordinates (x, y, z) normalized to 0-1 range
- Real-time tracking with ~30-60 FPS

### 2. Feature Normalization

Landmarks are normalized to be:
- **Translation-invariant**: Centered at the wrist
- **Scale-invariant**: Normalized by hand size
- **Rotation-robust**: Works from various angles

### 3. Gesture Classification

A 3-layer neural network classifies normalized landmarks into:
- 26 ASL letters (A-Z)
- Special gestures: space, delete

Model architecture:
```
Input (63 dims) → Dense(128) → Dropout(0.3) →
Dense(64) → Dropout(0.2) →
Dense(32) → Dropout(0.2) →
Output (28 classes)
```

### 4. Temporal Smoothing

Predictions are smoothed using a sliding window to:
- Reduce noise and flickering
- Only output consistent predictions
- Improve user experience

## Training Your Own Model

### Option 1: Quick Start (Pre-trained)

The application can work with gesture data from:
- [CRNN-based ASL Recognition](https://github.com/topics/asl-recognition)
- [MediaPipe Gesture Recognizer](https://developers.google.com/mediapipe/solutions/vision/gesture_recognizer)

### Option 2: Custom Training

1. **Collect Data**:
```bash
python scripts/collect_data.py --gesture A --samples 50
```

2. **Prepare Data**:
```bash
python scripts/prepare_data.py
```

3. **Train Model**:
```bash
python scripts/train_model.py --epochs 50 --batch-size 32
```

4. **Evaluate**:
```bash
jupyter notebook notebooks/ASL_Model_Optimization_90percent.ipynb
```

## Configuration

In the Streamlit sidebar, you can adjust:

- **Confidence Threshold**: Minimum confidence (0.0-1.0) to display predictions
- **Smoothing Window**: Number of frames to smooth over (1-10)
- **Show Landmarks**: Toggle visualization of hand landmarks

## Usage Tips

1. **Lighting**: Good lighting improves detection accuracy
2. **Hand Position**: Keep your whole hand visible and in frame
3. **Distance**: Optimal distance is about 30-60cm from camera
4. **Background**: Uncluttered backgrounds work best
5. **Speed**: Take pauses between gestures for better accuracy

## Troubleshooting

### Issue: Webcam not detected
```bash
# Check available cameras
python -c "import cv2; print(cv2.getBuildInformation())"
```

### Issue: Low FPS or lag
- Reduce frame resolution in settings
- Close background applications
- Disable landmark visualization

### Issue: Poor gesture recognition
- Collect more training data for struggling gestures
- Check lighting conditions
- Retrain with higher epochs (100+)

## Performance Metrics

- **Detection FPS**: 30-60 FPS on modern hardware
- **Model Accuracy**: ~95% on trained ASL alphabet (with sufficient training data)
- **Latency**: ~50-100ms per frame
